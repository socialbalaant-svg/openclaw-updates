🎯 ملخص الإصدار

سمعنا الملاحظات. يركّز الإصدار 2026.6.11 على الحوافّ الخشنة التي تجعل OpenClaw أقل اعتماداً، مع إصلاحات للردود في غير محلها، والإرسال العالق، وإعادة الاتصال، وفشل إعداد النماذج، وإعدادات افتراضية أكثر أماناً للمسؤول.

القنوات موثوقية التسليم عبر القنوات

إصلاحات شاملة في Telegram، WhatsApp، Matrix، Google Chat، iMessage، Feishu، Mattermost، WebChat، Control UI و TUI.

🎙️ Feishu تعرض مدة الصوت
ردود Feishu الصوتية من OpenClaw تعرض الآن مدة الصوت في فقاعة المحادثة، ليرى المستلم المدة قبل التشغيل.
🧹 Telegram: رد واحد لكل دور
محادثات Telegram المتابعة في WebChat تعرض الآن رد مساعد واحد لكل دور، وتحافظ على الردود اللاحقة في المحادثة النشطة بدلاً من تكرار الإجابات أو إرسالها مجدداً إلى Telegram.
🔄 WhatsApp Groups: سياق ثابت
محادثات جماعية WhatsApp تحافظ الآن على الرسالة الصحيحة وسياق المجموعة بشكل أكثر موثوقية أثناء إعادة المحاولة وإعادة الاتصال وتغييرات المجموعة.
🛡️ Matrix E2EE: لا مزيد من تسريب الذاكرة
بوابات Matrix E2EE يمكنها البقاء متصلة أثناء الاستخدام طويل الأمد بدلاً من استهلاك الذاكرة تدريجياً حتى الانهيار الذي يأخذ القنوات والأعمال الجارية معه.
📱 iMessage: تجميع ذكي
رسائل iMessage المكوّنة من أمر + رابط تبقى معاً كدور OpenClaw واحد عند وصول preview الرابط متأخراً، مع إبقاء الرسائل السريعة المنفصلة منفصلة لمن فعّل تجميع DM.

النماذج استرداد المزود والنماذج

🎤 MiniMax TTS: تنسيق صوتي صريح
نصوص MiniMax TTS والملاحظات الصوتية أقل عرضة للفشل لأن OpenClaw يطلب صراحةً تنسيق الصوت القابل لفك التشفير بدلاً من الاعتماد على الإعدادات الافتراضية للمزود.
📊 Logs: تفاصيل المزود والنموذج
يمكن لمشغلي البوابة رؤية تفاصيل المزود والنموذج وحالة الطلب والتوقيت في السجلات العادية، لتسهيل تشخيص مشاكل توجيه النماذج والنقل دون تفعيل debug logging إضافي.
🧹 System prompts نظيفة
النماذج عبر Google و Mistral و OpenAI Responses و Azure OpenAI Responses و ChatGPT/Codex Responses تتلقى تعليمات نظام نظيفة دون تسرّب marker حدود الكاش الداخلية لـ OpenClaw.
⚙️ Gemini Cron: إصلاح Schema
استدعاءات أداة Cron باستخدام نماذج Gemini عبر مزودين متوافقين مع OpenAI تعمل الآن دون أن تتسبب الحقول nullable في رفض schema.
🎯 تغيير النموذج الافتراضي فعّال فوراً
بعد تغيير النموذج الافتراضي، بدء جلسة قناة جديدة بـ `/new` أو `/reset` يستخدم النموذج الجديد بدلاً من إعادة استخدام النموذج المخزن مؤقتاً، مع بقاء overrides `/model` الصريحة كما هي.
🌐 Codex usage خلف Proxy
خلف HTTP/HTTPS proxies، فحوصات Codex/OpenAI usage وquota في `openclaw status --usage --json` و Control UI تسترد نوافذ الاستخدام بدلاً من الفشل عندما يكون chatgpt.com غير قابل للوصول مباشرة.

الذاكرة استمرارية الجلسات والذاكرة والثقة

💾 OpenAI Responses: استرداد المحادثة
محادثات الوكلاء المتأثرة باستخدام OpenAI Responses يمكنها الاسترداد والاستمرار في الرد بعد أن تترك استجابة القناة المرئية سجلها المحفوظ ناقصاً، بدلاً من فشل كل دور لاحق.
📝 Codex: حد حجم النص
عندما ينتج وكيل مدعوم بـ Codex مخرجات أدوات كبيرة بشكل غير عادي، تحافظ المحادثات المحفوظة والمعاد تشغيلها على نصها ضمن حد الحجم المعتاد مع إبقاء المحتوى غير النصي كما هو.
🔌 Control UI: استمرار بعد الانقطاع
محادثات Control UI تبقى مرئية وتستمر في نفس الجلسة بعد sleep أو انقطاع شبكة أو إعادة اتصال Gateway، بدلاً من الاختفاء عند إرسال الرسالة التالية.
📂 Codex & Copilot: نسخ متماثل دقيق
تكاملات Codex و Copilot المجمّعة تحافظ الآن على سجل المحادثة وتحديثات النسخ المتماثل مرتبطة بجلسة OpenClaw الصحيحة.
📚 Memory Wiki: ملاحظات المستخدم آمنة

إرشادات بدء التشغيل تحدد الآن الإدخال غير المطابق وتعرض plugin ids المكتشفة لتصحيح `plugins.allow`. تنبيهات ثقة الإضافة تتضمن مثالاً جاهزاً للنسخ. ترحيلات Codex تعمل مع التثبيتات العامة القياسية. التثبيتات والتحديثات تتعافى من dependency pins القديمة بدلاً من فشل `npm EOVERRIDE`.

الأمان Gateway والثقة

🔐 مسارات حزم محمية من lookalikes
مصادر حزم OpenClaw الموثوقة ترفض الآن مسارات sibling المشابهة، فالوثوق بـ `/artifactory/openclaw` لم يعد يسمح بمسارات مثل `/artifactory/openclaw-malicious`.
⏱️ Gateway probe: timeout محترم
عندما تكون بوابة بعيدة مُكوّنة أو صريحة بطيئة لكن قابلة للوصول، `openclaw gateway probe --timeout ...` ينتظر الـ timeout المطلوب بدلاً من الإبلاغ عنها كغير قابلة للوصول بعد قطع داخلي أقصر.
🖼️ صور Codex Remote تصل كمرفقات
الصور المولّدة من Codex app-server بعيد تصل الآن كمرفقات بدلاً من إظهار `Media failed` أو إرجاع نص فقط بعد التوليد الناجح.
📊 systemd: البقاء تحت ضغط الذاكرة
عندما يقتل Linux child command أو session بسبب ضغط الذاكرة، تبقى بوابات OpenClaw المُدارة بـ systemd قيد التشغيل وتحافظ على اتصالات القناة حية مع الإبلاغ عن فشل child.

الأدوات واجهات العميل والتشغيل

🖥️ Control UI: حالة فشل صريحة
عندما تفشل رسالة WebChat قبل بدء الوكيل، تعرض WebChat و Control UI الجلسة كفاشلة بدلاً من تركها تبدو لا تزال قيد التشغيل.
📱 iOS: ألوان موحدة
شاشات iOS تستخدم الآن ألوان accent وحالة OpenClaw متسقة عبر onboarding والإعدادات والمحادثة ومطالبات الموافقة وأذونات الصوت والـ widgets وطرق عرض المحادثة المشتركة.
📊 /status: سطر واحد للنموذج
`/status` يعرض الآن النموذج النشط وكيفية مسح اختيار مثبّت في سطر واحد مدمج، ليتمكن مستخدمو Discord وغيرهم من فحص حالة النموذج دون شرح متعدد الأسطر.
⚡ CLI Help أسرع 30x
مساعدة `doctor` و `gateway` و `models` و `plugins` و `sessions` و `tasks` تظهر الآن في عشرات الميلي ثانية، بينما كانت `sessions --help` و `tasks --help` تستغرق ~1.6-1.8 ثانية سابقاً.
🗓️ Cron: نماذج بديلة لكل job
وظائف cron المجدولة الفردية يمكنها الآن استخدام نماذج بديلة خاصة بها، أو التشغيل مع تعطيل البدائل، أو العودة إلى وراثة البدائل العادية عبر CLI بدلاً من تحرير payload منخفض المستوى.
🧹 Cron deleteAfterRun: تنظيف تلقائي
وظائف cron المعزولة باستخدام `deleteAfterRun` تزيل الآن جلستها المؤقتة وtranscript بعد الانتهاء، بما في ذلك التشغيل مع تعطيل delivery، مما يقلل الملفات القديمة.
